# 面试高频故障场景

四、面试高频故障场景
————————————————————————————————————————————————————————————————


场景一：服务器 CPU 飙到 100%

  排查命令：
  top                             找出最吃 CPU 的进程 PID
  top -p PID -H                   看该进程中哪个线程在消耗
  lsof -p PID                     看该进程打开了什么文件
  journalctl -u 服务名 --since "10 min ago"  查看业务日志

  面试话术：
  「先 top 定位 CPU 最高的进程。如果是业务进程，确认是否发版或流量高峰导致；
   如果是异常进程，先 kill -15 优雅停止，不行再 -9。
   然后排查启动脚本和定时任务，封堵后门。」


场景二：磁盘满但 du 找不到大文件

  排查命令：
  lsof | grep deleted             找已删除但仍被进程占用的"幽灵文件"
  看到 deleted 标识即问题所在
  systemctl restart 对应进程        重启进程释放空间

  面试话术：
  「文件已删除但进程还在写入，空间不会释放。
   用 lsof | grep deleted 找到占用进程，重启即可回收空间。」


场景三：端口不通

  排查命令：
  ss -tlnp | grep 端口号           服务在监听吗？
  firewall-cmd --list-all         防火墙放行了吗？
  telnet 127.0.0.1 端口号          本机通吗？
  telnet 公网IP 端口号             远端通吗？
  云服务器检查安全组

  面试话术：
  「逐层排查：服务监听 → 防火墙 → 云安全组。
   最常见原因是安全组忘记加规则。」


场景四：用户说上不了网

  排查命令：
  ipconfig                        查看 IP（169.254 开头即 DHCP 失败）
  ping 127.0.0.1                  网卡正常？
  ping 网关                       局域网通？
  ping 114.114.114.114            外网通？
  nslookup baidu.com              DNS 正常？

  面试话术：
  「从网卡到局域网到外网到 DNS 逐层排查。
   80% 桌面网络问题是 DNS 或网线松动。」


场景五：用户电脑蓝屏

  排查命令：
  1. 记蓝屏代码 → 搜索定位
  2. eventvwr.msc → 系统 → 事件 ID 1001
  3. 安全模式：设置 → 更新和安全 → 恢复 → 高级启动
  4. sfc /scannow    修复系统文件

  面试话术：
  「先记录错误代码搜索原因，查事件查看器找更多上下文。
   多由驱动或更新导致，进安全模式卸载即可。
   频繁蓝屏需排查内存或硬盘硬件。」


场景六：Docker 容器启动失败

  排查命令：
  docker logs 容器名                 看报错日志
  docker inspect 容器名 | grep -i ip  看网络配置
  docker exec -it 容器名 sh          进容器手动排查
  docker-compose logs -f            编排项目看全套日志

  面试话术：
  「先看 docker logs 定位具体报错。大部分是端口冲突或配置错误。
   进容器手动执行命令验证，日志清楚就能定位。」


场景七：桌面运维——新员工配电脑

  完整流程：
  硬件检查 → 加域 → 装必备软件（Office/企微/VPN/浏览器） → 配邮箱 → 配打印机 → 映射共享文件夹 → 资产登记

  面试话术：
  「有标准 SOP：拆箱检查 → 加域 → 装软件 → 配外设 → 登记资产。
   每台电脑交付状态一致，便于统一管理。」
